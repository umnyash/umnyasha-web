Флоаты.

Свойство float.

Изначально свойство float было предназначено, чтобы включать обтекание элементов текстом. Для отдельных элементов в потоке текста, врезок, блоков с заметками и ссылками обтекание подходит как нельзя лучше.

Значения:

  • left — элемент ужимается по ширине под своё содержимое, прижимается к левому краю родителя, другие элементы обтекают его справа.
  • right — элемент ужимается по ширине под своё содержимое, прижимается к правому краю родителя, другие элементы обтекают его слева.
  • none — (по умолчанию) отключает режим обтекания и возвращает элементу нормальное поведение.

Зафлоаченному элементу можно явно задавать размеры и отступы.

Если зафлоатить строчный элемент, то он начинает вести себя как блочный, а именно воспринимать размеры и отступы.

float и выпадание из потока.

Зафлоаченные элементы выпадают из потока, но лишь частично:

  • Блочные элементы, которые идут в коде после зафлоаченного блока, перестают его замечать. Они подтягиваются вверх и занимают его место, как будто его и нет, но текст внутри этих блочных элементов продолжает воспринимать зафлоаченный блок и обтекает его.
  • Строчные элементы, расположенные в коде после зафлоаченного блока, начинают обтекать его со свободной стороны.

Такое поведение флоатов даёт интересные эффекты:

  • Эффект прохождения сквозь блоки. Проявляется, когда зафлоаченный элемент выше, чем несколько последующих за ним блоков.
  • Эффект выпадания из родителя или схлопывание родителя. Проявляется тогда, когда все дочерние блоки в родителе зафлоачены. В этом случае родитель схлопывается по высоте, как будто в нём нет содержимого, а блоки выпадают из него.

Друг для друга флоатные элементы остаются видимыми. Идущие друг за другом флоаты выстраиваются в ряд, пока им хватает свободного места. Если места не хватает, то они начинают переноситься на следующую строчку. Почти как текст.

Как раз эта особенность флоатов позволила применять их для создания сеток. Ведь в начале 2000-х в CSS не было другого способа создавать колонки и задавать им размеры.

Свойство clear.

Свойство clear применяется ко флоатным и блочным элементам и запрещает им обтекать флоатные элементы, находящиеся с указанной от него стороны.

Значения:

  • left — запрещено обтекание элементов, находящихся слева.
  • right — запрещено обтекание элементов, находящихся справа.
  • inline-start — аналогично left для ltr и right — для rtl.
  • inline-end — аналогично right для ltr и left — для rtl.
  • both — запрещено обтекание элементов, находящихся с любой стороны.
  • none — (по умолчанию) обтекание разрешено.

Если после флоатного элемента расположен элемент с запрещённым обтеканием, то последний опускается под флоатный.

Свойство clear «учит» блочные элементы «видеть» зафлоаченные.

Обычная сетка на флоатах.

Сетки на флоатах обычно делаются так:

  • Создаём блок-контейнер для колонок.
  • В контейнер добавляем флоатные блоки-колонки.
  • Добавляем распорку.
  • Рассчитываем ширины колонок так, чтобы им хватало места в родителе.

При создании колонок на флоатах нужно помнить про переполнение. Для этого достаточно задавать ширину колонок. Если текст в колонке будет слишком длинным, он перенесётся на следующую строку, и это нормальное поведение. Но если забыть задать ширину, то колонка сместит вторую колонку вниз. И это недоработка верстальщика.

Распорка.

Чтобы предотвратить выпадание колонок и схлопывание контейнера, после зафлоаченных колонок нужно добавить пустой элемент-распорку со значением both свойства clear. Этот элемент видит колонки и не даёт пройти через себя, а заодно и растягивает родительский блок по высоте.

Для таких распорок прижилось специальное название класса — clearfix.

Пример:

<!-- HTML -->
<div class="container">                /* Блок-контейнер. */
  <div class="column-1"> ... </div>    /* Колонка, флоат. */
  <div class="column-2"> ... </div>    /* Колонка, флоат. */
  <div class="clearfix"></div>         /* Распорка. */
</div>

/* CSS */
.clearfix {
  clear: both;
}

Псевдораспорка.

Распорку можно создать с помощью псевдоэлемента.

Один из самых распространённых вариантов – это добавить класс clearfix контейнеру, внутри которого лежат флоатные колонки и в CSS добавить следующее правило:

.clearfix::after {
  content: "";
  display: table;
  clear: both;
}

После этого в контейнер не нужно добавлять дополнительный элемент-распорку, так как распорка создаётся внутри контейнера с помощью псевдоэлемента.

<div class="container clearfix">      /* Блок-контейнер. */
  <div class="column-1"> ... </div>    /* Колонка, флоат. */
  <div class="column-2"> ... </div>    /* Колонка, флоат. */
</div>

Префикс layout.

Использование значимых префиксов для именования классов — хороший приём, который нужно использовать в вёрстке.

Колонки имеют класс layout-column:

<div class="container clearfix">
  <div class="layout-column layout-column-1"> ... </div>
  <div class="layout-column layout-column-2"> ... </div>
</div>

Префикс layout обозначает «сетка» или «раскладка». Таким образом, мы даём понять, что классы с этим префиксом предназначены для создания сетки. Обычно для таких классов задают только флоаты, размеры и внешние отступы.
