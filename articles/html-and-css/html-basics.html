Основы HTML.

HTML (HyperText Markup Language) — язык гипертекстовой разметки.

Гипертекст — это документы, связанные между собой ссылками.

HTML отвечает за структуру и содержание страницы.

HTML-разметка страницы пишется по определённым правилам, которые описываются в специальных документах — спецификациях. Следование этим правилам называют семантикой.

Спецификация HTML Living Standard.

В процессе разметки страницы все содержательные сущности документа размечаются подходящими по смыслу тегами. Декоративные (не содержательные) сущности в разметку входить не должны.

Правильно выполненная разметка делает внутреннюю структуру прозрачной и читаемой как для самого верстальщика, так и для его коллег, и для поисковых роботов, и для инструментов, помогающих при работе с сайтами — ридеров, «читалок» и прочих.

Тип документа.

Каждый HTML-документ начинается с декларации типа документа (document type) или «доктайпа». Тип документа необходим, чтобы браузер мог определить версию HTML и правильно отобразить страницу.

Для старой версии HTML доктайп выглядел так:

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3c.org/TR/html4/strict.dtd">

А для современной версии HTML уже намного проще:

<!DOCTYPE html>

Теги.

HTML состоит из тегов.

Теги бывают парными и одиночными.

Одиночные теги состоят из имени тега, заключённого между знаками < («меньше») и > («больше»).

<тег>

У парных тегов, в отличие от одиночных, есть вторая половинка — закрывающий тег. В закрывающих тегах перед именем ставится символ / («слэш»).

<тег></тег>

Парные теги могут содержать текст и другие теги.

Не все теги можно вкладывать в другие теги. Например, <h1></h1> нельзя вкладывать в <p></p>.

У вложенных тегов всегда нужно следить за правильным порядком закрытия. Вложенный тег не может закрываться позже родительского.

Помимо этого теги имеют блочный или строчный тип бокса. Теги с блочным боксом занимают всю ширину экрана, а со строчным — только некоторую её часть в зависимости от своего размера.

Теги с блочным боксом нельзя вкладывать в теги со строчным.

Проверка валидности.

Валидатор W3C — специальный сервис для проверки (валидации) HTML-кода.

Он проверяет код на формальные, строгие правила: правильно ли закрыты теги, правильно ли они вложены и так далее.

Классификация тегов.

Существует два основных типа тегов:

  • Metadata — мета-теги
  • Flow — поточные элементы (все теги, кроме метагегов)

Внутри Flow выделяют подкатегории:

  • Heading — заголовочные теги
  • Sectioning — теги для крупных смысловых разделов
  • Phrasing — фразовые теги
  • Embedded — встраиваемый контент (аудио, изображения, видео и т.д.)
  • Interactive — интерактивные элементы (кнопки, ссылки)

Элементы могут относиться к нескольким категориям.

Атрибуты тегов.

Значение атрибута указывается в кавычках после знака = («равно»).

У тегов может быть один или несколько атрибутов. Если атрибутов несколько, то они пишутся через пробел.

<тег атрибут1="значение" атрибут2="значение">

Комментарии.

Код, заключённый между символами <!-- и --> считается комментарием и работать не будет. Если эти символы удалить, то код заработает, это называется «раскомментировать».

<!-- Это комментарий в HTML -->

С помощью комментариев обычно временно отключают какой-то код или оставляют подсказки и разъяснения.

Основные теги HTML.

Простейшая HTML-страница состоит как минимум из 3 тегов: <html></html>, <head></head> и <body></body>.

<!DOCTYPE html>
<html>
  <head></head>
  <body></body>
</html>

Тег htlm.

Тег <html></html> располагается в документе сразу после доктайпа и содержит все остальные теги, включая <head></head> и <body></body>.

У тега <html></html> есть важный атрибут — lang, в котором задаётся язык документа.

Примеры:

<html lang="ru"></html> <!-- Русский язык -->
<html lang="en"></html> <!-- Английский язык -->

Тег head, служебная информация о странице.

Тег <head></head> предназначен для хранения служебной информации о странице, Он располагается первым в теге <html></html> сразу перед <body></body>.

Внутри <head></head> обычно содержится заголовок, ключевые слова, описание страницы и другие служебные данные. Также внутри него подключаются внешние ресурсы, например, стили. Содержимое этого тега не отображается на странице напрямую.

Тег body, содержимое страницы.

В теге <body></body> хранится содержимое страницы, которое отображается в окне браузера. Все тексты и картинки добавляются внутрь <body></body>.

Методика написания семантической разметки.

  1. Выделение крупных смысловых блоков на каждой странице сайта.
  2. Разметка в блоках крупных смысловых разделов.
  3. Выделение заголовка всего документа и заголовков смысловых разделов.
  4. Разметка мелких элементов в смысловых разделах.
  5. Разметка фразовых элементов.
