Обработка ошибок и исключений

Ошибки

Когда возникает ошибка, JavaScript генерирует объект, содержащий её детали.

В JavaScript есть множество встроенных конструкторов для стандартных ошибок:

  • new Error(сообщение)
  • new SyntaxError(сообщение)
  • new ReferenceError(сообщение)
  • new TypeError(сообщение)

Для встроенных ошибок, свойство name — это в точности имя конструктора, а свойство message берётся из аргумента.

Оператор "throw"

Оператор throw генерирует исключение.

throw значение

Значением исключения может быть что угодно, но всё же лучше, чтобы это был объект, желательно со свойствами name и message (для совместимости со встроенными ошибками).

"try..catch"

Конструкция try...catch позволяет ловить ошибки и исключения и предотвратить остановку выполнения программы при возможности их обработать.

try {
  // исполняемый код
  // если случится ошибка, управление сразу же перейдёт в блок catch
} catch (err) {
  // в параметр блока catch передаётся ошибка или значение исключения
} finally {
  // выполняется всегда после try/catch
}

Блок catch должен обрабатывать только те ошибки и исключения, которые ему известны и «пробрасывать» все остальные.

Секций catch или finally может не быть, то есть более короткие конструкции try..catch и try..finally также корректны.
