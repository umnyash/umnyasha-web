Основы анимации

Применение анимаций в вебе

Задача дизайнера – создать продукт, с помощью которого пользователь сможет эффективно решить свою задачуи получить положительный пользовательский опыт.

  • Обратная связь, реакции на действия пользователя.
  • Донесение информации о проиходящем. Например, для визуализации прогресса при выполнении какой-либо долгой операции.
  • Управление вниманием. Чтобы привлечь внимание пользователя к уведомлению или блоку, требующему от пользователя совершения действия.
  • Изменение состояний интерфейса. Порой сложно понять, куда делись какие-то компоненты при изменении сетки страницы и что к чему относится. Если сделать изменение структуры страницы анимированным, пользователь увидит откуда появились или куда передвинулись элементы.
  • Презентации. Есть вещи, которые сложно показать в виде статических картинок.
  • Сторителлинг – анимационные истории.
  • Декоративные анимации.

Анимируемые свойства

Анимировать можно только некоторые CSS-свойства. В основном, это свойства, изменяющие размер, цвет и позицию элементов.

Полный перечень анимируемых свойств и характер их анимирования

CSS transition

CSS transition (CSS переходы) позволяют плавно менять исходное значение CSS-свойства на новое вместо мгновенного переключения.

  • transition-property – анимируемое свойство

    Значения:

    • одно или несколько css-свойств, перечисленных через запятую
    • all — (значение по умолчанию) все анимируемые свойства
    • none — не применять анимацию ни к чему
  • transition-duration — длительность анимации (s, ms)
  • transition-timing-function — функция изменения скорости анимации
  • transition-delay — задержка перед началом анимации (s, ms)
  • transition — составное свойство (property duration timing-function delay)

Чтобы обозначить плавный переход, достаточно задать его длительность.

Для разных свойств можно задавать разные параметры перехода. Для этого параметры нужно перечислить через запятую в том же порядке, что и свойства:

transition-property: width, height;
transition-duration: 1s, 5s; /* Ширина меняется за 1 секунду, высота – за 5. */

CSS Animation

CSS-анимации имеют больше возможностей и помогают сделать то, что не могут CSS-переходы. В том числе создавать анимации с количеством шагов больше двух или запускаться сразу же при загрузке страницы и повторяться любое количество раз без какого-либо действия пользователя.

Создание анимации

Анимация создаётся с помощью правила @keyframes, в котором нужно задать название и описать в ключевыех кадрах изменяемые CSS-свойства.

Требования к названию:

  • можно использовать только латинские буквы, цифры, знаки дефиса (-) и нижнего подчёркивания (_)
  • нельзя ставить два дефиса подряд в начале имени
  • первый символ, или первый после дефиса, должен быть буквой
  • нельзя использовать специальные слова вроде inherit, none, initial, unset и т.п.

Ключевые кадры задаются в процентах от времени анимации. Значения 0% и 100% могут быть заменены ключевыми словами from и to.

Пример:

@keyframes coloring { /* Эта анимация называется "coloring". */
  0% { color: red; }
  33% { color: yellow; }
  66% { color: green; }
  100% { color: blue; }
}

Если начальный или конечный кадр не задан, то он автоматически создаётся, принимая исходное стилевое состояние элемента.

Задавать ключевые кадры можно в любом порядке. Это может быть удобно, чтобы сгруппировать одинаковые кадры. В таком случае их можно перечислить через запятую:

@keyframes example {
  0%, 100% { ... }
  10%, 30%, 50%, 70%, 90% { ... }
  20%, 40%, 60%, 80% { ... }
}

Применение анимации

  • animation-name — название
  • animation-duration — длительность (s, ms)
  • animation-timing-function — функция изменения скорости анимации
  • animation-delay — задержка перед началом анимации (s, ms)
  • animation-iteration-count – количество итераций

    Значения:

    • 0 — анимация не выполнится ни разу
    • целые положительные числа (по умолчанию 1)
    • infinite — анимация будет выполняться бесконечно
  • animation-direction – направление

    Значения:

    • normal — (по умолчанию) от начала к концу
    • reverse — от конца к началу
    • alternate — нечётные итерации выполняются в прямом направлении, чётные – в обратном
    • alternate-reverse — нечётные итерации выполняются в обратном направлении, чётные – в прямом
  • animation-fill-mode – применение стилей до и после анимации

    Значения:

    • none — (по умолчанию) не применять
    • backwards — применить стили первого кадра до начала анимации. Это заметно, если анимация имеет задержку
    • forwards — применить стили последнего кадра по завершении анимации
    • both — совмещает значения backwards и forwards
  • animation-play-state – состояние

    Значения:

    • running — выполняется
    • paused — приостановлена
  • animation — составное свойство (name duration timing-function delay iteration-count direction fill-mode play-state)

Чтобы применить анимацию к элементу, достаточно задать её имя и длительность.

Одному элементу можно задать несколько анимаций.

Если разными анимациями одновременно должно изменяется одно и то же свойство, то оно будет изменяться в соответствии с анимацией, заданной позже других.

Функции изменения скорости анимации

Свойства transition-timing-function и animation-timing-function в качестве значения могут принимать функцию steps, функции кубических кривых Безье, или их «псевдонимы».

Функции кубических кривых Безье

cubic-bezier(x1, y1, x2, y2)

Значения x и y – это координаты точек кривых на графике. При этом верными считаются значения x только в диапазоне от 0 до 1.

Например:

cubic-bezier(0, 0, 1, 1) /* linear */
cubic-bezier(0.42, 0, 1, 1) /* ease */

Псевдонимы:

  • linear — переход равномерный, без ускорений и замедлений.
  • ease — начинается медленно, затем ускоряется и к концу движения опять замедляется.
  • ease-in — начинается медленно, а к концу ускоряется.
  • ease-out — начинается быстро, а к концу замедляется.
  • ease-in-out — похоже на ease, то есть переход начинается и заканчивается медленно, но происходит это чуть интенсивнее.

Функция steps

При использовании функции steps(число_шагов, start/end) анимация будет происходить пошагово.

Первый аргумент обязателен. Длительность анимации делится на равные отрезки времени в соответствии с количеством шагов. Второй параметр влияет на то, когда в пределах каждого отрезка времени будет происходить изменение CSS-свойств: в начале или в конце. Значение end — по умолчанию.

При значении start анимация фактически будет завершаться чуть раньше.

Псевдонимы:

  • step-start — равнозначно steps(1, start). Стилевые свойства сразу принимают конечные значения.
  • step-end — равнозначно steps(1, end). Стилевые свойства находятся в начальном значении заданное время, затем сразу принимают конечные значения.
